编码器细节：3D Res2Net的改造和使用

核心思想

我们选用了一个叫Res2Net的”高手”来做编码器，但它本来是处理2D普通照片的。我们通过一些巧妙的”改造”，让它能处理我们的3D医学图像，并且还让它继承了之前在普通照片上学到的”经验”。

为什么选择Res2Net？

Res2Net的每个残差块内部都有分层式的残差连接，这让它能更有效地在粒度级别使用多尺度信息，扩展每一层的感受野。

就像工人同时拿着多种不同倍数的放大镜在工作：

  • 一个小放大镜看细微的纹理和边缘（小尺度特征）
  • 一个中倍放大镜看局部的结构（中尺度特征）
  • 一个大倍放大镜看大致的轮廓（大尺度特征）

这对于边界模糊的前列腺图像来说，非常有用。

编码器的具体工作流程

第一步：初始卷积（Convolution Stem）

给定的图像首先通过一个卷积茎，它由并行的3×3×3卷积和1×1×1卷积组成，后面跟着求和操作。

  • 3×3×3卷积：捕捉图像中小范围的、基本的局部特征（比如某个点是不是边缘）
  • 1×1×1卷积：主要负责调整数据的通道数，为后续处理做准备
  • 最后把两个结果相加，得到一个包含丰富基础信息的特征图

第二步：核心特征提取

然后依次进入四个Res2Net残差块，生成具有不同空间分辨率的特征。

预备层的输出会依次通过4个串联的Res2Net模块。每经过一个模块，特征图的空间尺寸会变小（下采样），但抽象程度和通道数会增加。这样，我们就得到了4个不同尺度的特征图，它们分别包含了从”细致”到”概括”的信息，为后续解码和融合做准备。

最关键的部分：如何从2D改造到3D？

改造方法一：第一个层

  • 我们首先将第一层中的7×7 2D卷积替换为7×7×3 3D卷积
  • 具体来说，具有单通道3D输入的7×7×3核等效于Res2Net中具有三通道2D输入的7×7核

改造方法二：其他层

  • 在其他层中，x×x的2D卷积直接转换为x×x×1的3D卷积

为什么要用”预训练”模型？

我们的编码器可以使用预训练的Res2Net进行初始化…这种初始化可以帮助迁移在大规模自然图像数据集上获得的图像表示能力。

“预训练”就是这个Res2Net”高手”已经在一个包含上百万张普通照片（ImageNet数据集）的”大型综合学校”里毕业了，学会了如何识别各种物体的基本特征（如边缘、形状、纹理）。

好处：我们不做”从零开始”的训练，而是让网络带着这些”基础知识”来学习我们的前列腺MR图像。这就像让一个已经学会”看图”的学生去专门学”看医学影像”，他会上手更快，效果更好。

Res2Net通道数增长机制

通道是什么？

在卷积神经网络中，一个”通道”可以理解为模型学习到的一种”特征探测器”或一种”专业的描述角度”。

  • 在输入层，对于灰度医学图像，只有1个通道，它代表原始像素的亮度
  • 在网络深层，一个通道可能专门负责探测边缘，另一个通道专门负责探测特定纹理，第三个通道可能专门识别圆弧状结构（可能是前列腺的轮廓）

通道数增加，就意味着模型有了更多”专业的描述角度”来理解和表示图像。

为什么需要增加通道数？

这背后是”用信息的广度（通道数），来弥补信息的深度（空间尺寸）的损失”的权衡策略。

典型的编码器下采样过程：

层级 空间尺寸（示例） 通道数（示例） 发生了什么？
输入 128 x 128 x 128 1 原始3D图像，只有亮度信息
第1层后 64 x 64 x 64 64 下采样，空间细节丢失了。但我们生成了64种不同的”基础特征”来描述这个更小的图
第2层后 32 x 32 x 32 128 尺寸再缩小一半。为了不丢失太多信息，我们将通道数翻倍
第3层后 16 x 16 x 16 256 尺寸再次减半。我们继续增加通道数到256
第4层后 8 x 8 x 8 512 到达最底层。空间信息最少，但此时我们有512个高度抽象和专业的”概念”

核心思想：当网络进行下采样时，特征图的空间尺寸会缩小，这意味着我们牺牲了精确的定位信息。为了补偿这种损失，我们增加通道数。这样，虽然每个位置的空间信息变粗糙了，但每个位置都由更多种类的特征来描述，从而保留了甚至增强了其信息含量。

在Res2Net中，这是如何具体实现的？

在MM-UNet的编码器中，通道数的增加并不是在Res2Net模块内部完成的，而是在进入下一个Res2Net模块之前，通过一个下采样卷积层实现的。

具体流程：

  1. 输入进入第一个Res2Net模块，假设输入通道数为64
  2. 在这个Res2Net模块内部，特征图会被分组处理，但最终输出的通道数通常保持不变（还是64）
  3. 当需要进入下一个、更深的Res2Net模块时，我们会先通过一个过渡层。这个过渡层通常是一个步长为2的3D卷积
    • 步长为2：实现了下采样，将特征图尺寸减半
    • 卷积核的输出通道数设置为128：这直接将通道数翻倍

所以，完整的流程是：
[当前层] -> Res2Net块（提炼特征，通道数不变） -> 下采样卷积（尺寸减半，通道数翻倍） -> [下一层]

初始卷积茎与Res2Net的分工

为什么要有初始卷积茎？

我们可以把这个过程想象成一个加工流水线：

工位1：初始卷积茎 (3x3x3 + 1x1x1)

  • 角色：初级加工车间
  • 任务：对”原始材料”（原始像素）进行最基础、最通用的初步处理
  • 具体工作：
    • 3×3×3卷积：识别出最基本的”构件”，如边缘、角点、斑点
    • 1×1×1卷积：将这些基础构件进行初步的整理和分类
  • 产出：一套质量更高、更有组织性的”半成品零部件”

工位2及以后：Res2Net模块

  • 角色：精密装配车间
  • 任务：接收来自”初级加工车间”的”半成品零部件”，进行复杂、精细的组装和多尺度分析
  • 具体工作：将这些基础特征组合成更复杂的模式
  • 产出：高度复杂和抽象的特征

为什么不能跳过”工位1”，直接让Res2Net处理原始像素？

理论上可以，但效率低下，效果可能不好。原因如下：

  1. 计算效率低：Res2Net模块结构复杂，计算成本高
  2. 优化难度大：让一个复杂的模块同时学习从极低层级到极高层级的特征，训练过程会非常困难和不稳定
  3. 分工明确，各司其职：
    • 初始卷积茎专门负责低级特征提取
    • Res2Net专门负责中高级特征提取和多尺度融合

总结

它们不是重复劳动，而是流水线上的前后工序：

  • 初始卷积茎 (3x3x3 + 1x1x1)：是特征提取的”启动器”或”基础层”
  • Res2Net：是特征提取的”主力军”或”增强层”

初始卷积茎为Res2Net提供了更好的”工作素材”，让Res2Net能够发挥出最大的效能。
